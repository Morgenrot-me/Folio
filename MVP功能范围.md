# 智能图像分类工具 — MVP 功能范围

> 创建时间：2026-03-24
> 版本：v0.1

---

## MVP 核心原则

> **做少，做稳，做出核心体验。**
>
> MVP 只需证明一件事：**"用 AI 特征驱动文件夹分类"这件事在端侧是可行且流畅的。**
> 手动整理（Slidebox 模式）作为独立模块，MVP 阶段不纳入。

---

## 一、纳入 MVP 的功能

### 1. 图片索引与特征提取（后台）
- [ ] 首次启动时扫描系统相册（MediaStore）
- [ ] 后台（WorkManager）逐张提取特征，充电+Wifi 条件下静默运行
- [ ] 提取内容：语义向量（MobileCLIP）、截图检测、文字检测（ML Kit）、模糊度、颜色调性、pHash
- [ ] 进度持久化：中断后可续接，不重复处理已索引图片
- [ ] 首页展示索引进度（已处理张数 / 总张数）

### 2. 智能文件夹（核心功能）
- [ ] 创建智能文件夹（名称、图标、颜色）
- [ ] 规则编辑器：支持 AND / OR / NOT 逻辑组合
- [ ] MVP 支持的 LEAF 特征类型（6种）：
  - `IS_SCREENSHOT`：是否为截图
  - `HAS_TEXT`：是否含文字
  - `BLUR_SCORE`：模糊度阈值
  - `COLOR_WARMTH`：色温范围
  - `SEMANTIC_LABEL`：语义标签（从预设标签集选择）
  - `CLUSTER`：属于某聚类
- [ ] 规则变更后自动重新匹配，更新文件夹内容
- [ ] 浏览智能文件夹内的图片（网格视图）

### 3. 相似图片聚类
- [ ] pHash 近似重复图片检测
- [ ] 基于语义向量的 HDBSCAN 聚类（后台运行）
- [ ] 聚类结果展示（每组显示代表图）
- [ ] 用户可为聚类命名

### 4. 图片详情页
- [ ] 显示单张图片的所有已提取特征（可视化）
- [ ] 显示该图所属的全部智能文件夹
- [ ] 语义向量的人类可读标签展示（Top-5 标签 + 置信度）

### 5. 基础权限与设置
- [ ] 首次启动引导（申请 `READ_MEDIA_IMAGES` 权限）
- [ ] 手动触发"重新扫描"（处理删除/新增的图片）
- [ ] 索引数据库本地存储，卸载前数据可导出（可选）

## 二、MVP 不纳入的功能（后续版本）

| 功能 | 理由 |
|------|------|
| Slidebox 手动整理模式 | 独立模块，用户已有独立项目构图，另行开发 |
| 物理主文件夹移动（导出） | 涉及 SAF 文件操作，MVP 先稳定虚拟文件夹 |
| GPS 区域规则（`GPS_REGION`） | 依赖 EXIF 地理数据，覆盖率不稳定 |
| 相似图搜索（`SIMILARITY`） | 需要向量 ANN 检索，MVP 用暴力余弦相似度即可，但作为规则类型暂不开放 |
| AI 联动（LLM 规则建议） | 二期功能 |
| 桌面端适配 | 安卓 MVP 稳定后再做 |

## 三、MVP 技术里程碑

```
M1 — 能跑起来
  ├─ Flutter 项目初始化
  ├─ Room/SQLite 数据库表创建（drift ORM）
  └─ MediaStore 图片扫描读取

M2 — 特征提取跑通
  ├─ MobileCLIP 模型集成（TFLite）
  ├─ ML Kit 文字检测集成
  ├─ 截图/模糊/颜色算法实现
  └─ WorkManager 后台调度建库

M3 — 智能文件夹可用
  ├─ 规则树数据结构实现
  ├─ RuleEvaluator 规则引擎
  ├─ 规则编辑器 UI
  └─ 文件夹内图片网格展示

M4 — 聚类可用
  ├─ pHash 近似重复检测
  ├─ HDBSCAN 聚类（Dart 实现或 native 插件）
  └─ 聚类展示 UI

M5 — MVP 完整体验
  ├─ 图片详情页（特征可视化）
  ├─ 首次启动引导
  └─ 性能调优（目标：中端机索引 1000 张 < 3 分钟）
```

---

## 四、成功标准

MVP 认为成功，需满足：
1. 在真实设备上，1000 张图片后台索引完成时长 < 5 分钟
2. 用户能创建一个包含 AND/OR/NOT 逻辑的智能文件夹，并看到正确匹配结果
3. App 冷启动时间 < 2 秒
4. 规则匹配查询响应时间 < 500ms（针对 1 万张图）
