# 工具脚本架构说明

## 1. 目录作用
存放维护项目运转的周边开发脚本，包含环境检查、批量启动、持续集成配置(CI/CD)与清理工具等。

## 2. 文件列表
- `download_model.py` [NEW]: 自动化模型装载程序。通过 `urllib` 直连 Huggingface 库，拉取最契合目前 App 张量结构的 MobileCLIP S2 权重，并自动安放至 `frontend/assets/models/` 目录中。

## 3. 技术说明
支持各类以 Bash、Python 编写的自动化脚本，用以提升开发与部署效率。

## 4. 使用示例
举例：可创建 `setup.sh` 或 `init.bat` 脚本执行项目环境安装。

## 5. 扩展方式
- 新增脚本优先放在根级或按类别切分子目录。

## 6. 注意事项
- 所有的 Python 或 Bash 代码内需完整注释其功能（特别是若有提示类执行需要用户同意时）。
- 执行涉及危险权限操作脚本时须预留确认提示。
- 每新增脚本立刻更新本说明。
