# AI 识别管线设计说明

本文档说明智能图库（Folio）采用的三条 AI 识别管线的设计思路、各自优势和协作方式。

---

## 为什么需要三条管线？

传统移动端相册 AI 通常只使用一个通用分类模型（如 MobileNetV1-ImageNet1000），但其存在固有缺陷：

| 问题 | 例子 |
|------|------|
| 粒度过细，不实用 | 知道是「威玛猎犬」但不知道是「狗」 |
| 场景识别缺失 | 无法区分「海滩」和「山顶」 |
| 人像/食物覆盖差 | ImageNet-1000 无通用「人」「食物」类别 |
| 向量质量有限 | MobileNetV1 向量不适合跨语义相似搜索 |

三条专项管线分工合作，覆盖单一模型无法兼顾的全部维度。

---

## 管线一：ML Kit ImageLabeler（通用物体标签）

**定位**：告诉用户「这张照片里有什么」

| 属性 | 值 |
|------|-----|
| 提供方 | Google ML Kit（bundled on-device） |
| 模型大小 | +3 MB（打包进 APK） |
| 标签数量 | ~400 个常用类别 |
| 置信度阈值 | 0.65（过滤低置信噪音） |
| 输出示例 | 狗 · 动物 · 户外 · 草地 |
| 联网要求 | ❌ 完全离线，不依赖 Play Services |
| 实现状态 | ✅ Phase 1 已完成 |

**擅长识别**：
- 动物（狗 / 猫 / 鸟 / 鱼…）
- 人物（人 / 自拍 / 儿童 / 人群…）
- 食物（食物 / 水果 / 蔬菜 / 甜品…）
- 日常物品（汽车 / 植物 / 建筑 / 电子设备…）
- 运动与活动（运动 / 舞蹈 / 音乐…）

**不擅长**：具体场景地点（停车场 / 卧室 / 海滩 / 餐厅）→ 交给 Places365

---

## 管线二：Places365（场景地点标签）

**定位**：告诉用户「这张照片是在哪里拍的」

| 属性 | 值 |
|------|-----|
| 提供方 | MIT Computer Science 实验室 |
| 模型大小 | ~5-10 MB（TFLite INT8 量化） |
| 标签数量 | 365 个场景类别 |
| 训练数据 | 180 万张场景照片，专为「人拍的照片」设计 |
| 输出示例 | 海滩 · 海岸 |
| 联网要求 | ❌ 完全离线 |
| 实现状态 | 🔄 转换脚本就绪，待集成 |

**擅长识别**（部分类别示例）：

| 室内 | 室外自然 | 室外城市 |
|------|---------|---------|
| 卧室 / 客厅 | 海滩 / 山顶 | 城市街道 / 停车场 |
| 厨房 / 浴室 | 森林 / 草地 | 桥梁 / 广场 |
| 餐厅 / 咖啡厅 | 河流 / 湖泊 | 建筑工地 / 机场 |
| 办公室 / 教室 | 沙漠 / 雪地 | 商场 / 图书馆 |

**与 ML Kit 互补**：ML Kit 说「动物 · 狗」，Places365 说「海滩」→ 合并结果：「狗 · 动物 · 海滩」

## 管线三：MobileCLIP（语义向量 + 零样本理解）

**定位**：提供高质量图片语义嵌入，支持「按内容搜索」

| 属性 | 值 |
|------|-----|
| 提供方 | Apple Research（2024） |
| 模型大小 | ~22 MB（MobileCLIP-S0 TFLite） |
| 输出维度 | 512 维 float32 向量 |
| 核心能力 | 视觉-语言对齐，零样本分类 |
| 联网要求 | ❌ 完全离线 |
| 实现状态 | ⏳ Phase 2，代码槽位已预留 |

**相比 MobileNetV1 向量的优势**：
- MobileNetV1 向量 = ImageNet 类别的概率分布，语义贫乏
- MobileCLIP 向量 = 与人类自然语言对齐的语义空间，理解更深
- 同样场景的图片，MobileCLIP 向量距离更近（聚类质量更好）
- 支持后续「文本搜索图片」功能（只需计算文本向量与图片向量的余弦相似度）

## 三管线协作流程

```
                    ┌── ML Kit ImageLabeler ──→ 狗 · 动物 · 户外    ┐
                    │                                               │
图片                ┼── Places365 TFLite ─────→ 海滩 · 海岸        ├──→ tags = "狗,动物,户外,海滩"
                    │                                               │
                    └── MobileCLIP TFLite ────→ 512 维语义向量 ────→ semanticVector（聚类用）
```

**写库字段映射**：

| 数据库字段 | 来源 |
|-----------|------|
| `tags` | ML Kit 标签 + Places365 标签（合并去重） |
| `semanticVector` | MobileCLIP 512 维向量（BLOB） |
| `blurScore` | 拉普拉斯方差算法（不依赖任何模型） |
| `colorWarmth` / `dominantHue` | RGB 均值算法（不依赖任何模型） |
| `hasText` | ML Kit OCR TextRecognizer |
| `isScreenshot` | 文件路径关键字匹配 |
