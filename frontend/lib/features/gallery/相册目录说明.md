# 相册目录说明 (Gallery)

## 1. 目录作用
`gallery/` 目录主要负责呈现照片时光轴大盘流以及单张大图的超级详情浏览。此模块深度交互并极大考验设备的渲染堆叠与滑动帧率优化指标。

## 2. 文件列表
- `gallery_screen.dart`: **本目录下负责瀑布流视觉呈现的核心**。它使用了原生 iOS/Android 的高级嵌套视图范式（`SliverToBoxAdapter` 混合 `SliverGrid`）外加全屏滚动拦截抓手来保证海量大图能在一瞬间空降穿梭，且按纯文字日期分隔标题聚合。
- `image_detail_screen.dart`: 负责基于 `Hero` 动画飞跃转场进入的一套满屏横滑大图查看器，其底层整合了对于 `images` 数据表的所有单项管理闭环、大图解析复用及 AI 特征雷达属性读取。

## 3. 技术说明
在 `gallery_screen.dart` 的设计中，曾遭遇过“底层高分级更新事件反噬”问题。由于 `drift` 的 `.watch()` 会因后台 AI 任务执行频繁导致主列表不停刷新重建，本层级已经**果断斩断了对数据库更新流的强绑定引用**。而是采用“分页状态流 (Pagination)”并在下拉距底 500 像素时预提取 (Pre-fetch)。这使得主线程在面临疯狂的数据入库时依旧不受微毫影响。

## 4. 使用示例
当系统主页大盘想要打开历史回溯相册库时，经 Navigator 路由推开。
```dart
Navigator.push(context, MaterialPageRoute(builder: (_) => const GalleryScreen()));
```

## 5. 扩展方式
如果想要基于地理位置经纬度实现一套地图聚落功能视图，可以在该文件夹下派生 `map_gallery_screen.dart` 并复用底层的单向抓取流逻辑。

## 6. 注意事项
请特别警惕，此处使用的 `Image.file()` 被强制附加上了 `cacheWidth: 300` 参数。**绝对不要移除该参数**！一旦移除，滑动时光轴时将会触发 Flutter 系统级超高分辨率原图完整内存解压从而瞬间冲垮设备运存 (RAM) 导致系统强杀 (`OOM`) 闪退。
