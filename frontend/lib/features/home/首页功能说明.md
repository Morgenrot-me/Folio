# 主页目录说明 (Home)

## 1. 目录作用
`home/` 目录主要负责 App 启动后的主视觉（Dashboard）呈现。大盘页面汇集了从底层查库得到的核心数据统计、实时任务进度以及最新的媒体功能快捷入口。

## 2. 文件列表
- `home_screen.dart`: **本目录下唯一且最核心的文件**。它组合了流监听（StreamBuilder）、智能后台队列发派以及相册入库 UI 反馈。

## 3. 技术说明
在最近的底层管线升级中，`home_screen.dart` 的 `_handleScan` 触发逻辑不再仅单一依赖受设备省电机制钳制的 `WorkManager`（容易导致“入库完成后迟迟不分析”），而是使用双路并发：
```dart
await BackgroundAiWorker.schedule(); // 注册 Android 锁屏保活系统后台排队（作为兜底）
scanner.analyzeUnanalyzedImages(); // 即时从当前 Isolate 前台抢占调度，让用户肉眼可见 AI 进度！
```

## 4. 使用示例
它是应用根路由进入的第一个画面，承载着相册权限请求与核心数据的挂载：
```dart
// 在 MaterialApp 顶层直接加载并监听 Provider
home: const HomeScreen(),
```

## 5. 扩展方式
如果需要在此大盘继续增加比如“内存占比”、“今日耗电”等其余图表，请在该目录下拆分出 `home_widgets.dart`，避免将状态管理嵌套到 400 行以上的层级中。

## 6. 注意事项
此处订阅了多种 `drift` 的实时响应数据流（如 `database.watchAnalyzedImagesCount()`）。严禁在此页面增加过多高频全量数据的重建以免引发首页掉帧。每一次扫描和清理孤儿文件的过程都要确保正确触发了主存的状态清理。
