# folders/ — 智能文件夹功能目录

## 目录作用

实现「智能分类」标签页的核心 UI，包括文件夹列表、新建向导、文件夹详情三个页面。
智能文件夹是 Folio 的核心特性：用户配置规则树，系统自动将符合条件的图片归入对应文件夹。

---

## 文件列表

| 文件名 | 作用 |
|--------|------|
| `folders_screen.dart` | 智能文件夹列表页（首页Tab），展示所有已创建的文件夹卡片，每卡显示匹配图片数。支持：手动触发规则重匹配（AppBar 刷新按钮）、文件夹重命名（弹出 TextField 对话框）、删除确认弹窗 |
| `create_folder_screen.dart` | 新建智能文件夹向导页，支持配置四类规则：1) 排除截图（IS_SCREENSHOT）、2) 仅含文字（HAS_TEXT）、3) 清晰度阈值（BLUR_SCORE，Slider）、4) AI 语义标签多选（SEMANTIC_LABEL，FilterChip）。保存时自动生成完整规则树（AND 根节点 + OR 子节点 + LEAF 叶节点）并写入数据库 |
| `folder_detail_screen.dart` | 文件夹详情页，展示该文件夹内所有 AI 匹配图片的 3 列缩略图网格，支持点击进入图片详情页（带 Hero 过渡动画） |

## 规则树结构说明

新建文件夹时写入的规则树示例（用户选了「排除截图」且选了「海边」「山」两个标签）：

```
SmartFolder
└── rootRuleId → AND（根节点）
    ├── LEAF: IS_SCREENSHOT == false
    └── OR（AI 标签组）
        ├── LEAF: SEMANTIC_LABEL CONTAINS "海边"
        └── LEAF: SEMANTIC_LABEL CONTAINS "山"
```

规则评估由 `SmartFolderMatcherService` 负责（见 `core/services/`）。

---

## 技术说明

- **实时图片数量**：`FoldersScreen` 通过 `AppDatabase.watchFolderImageCounts()` 流自动更新每个文件夹的匹配图片数，无需手动刷新
- **重命名实现**：`_renameFolder` → `_doRename`，使用 `drift.Value` 部分更新 `smartFolders` 表中的 `name` 字段
- **AI 标签来源**：场景标签（31 个）来自 Places365 模型输出，内容标签（24 个）来自 ML Kit ImageLabeler 输出，均已存入 `images.tags` 字段（逗号分隔）

---

## 扩展方式

- **新增标签分组**：在 `create_folder_screen.dart` 顶部的 `_kSceneTags` / `_kContentTags` 常量列表中追加词条即可
- **新增规则类型**：在 `_saveFolder` 方法中仿照现有 LEAF 节点模式添加新 `featureType`，并在 `SmartFolderMatcherService._evaluateLeaf` 中添加对应的 `case`
- **颜色/图标选择**：`SmartFolders` 表已有 `icon` 和 `color` 字段，可在 `create_folder_screen` 中添加 UI 选择器

---

## 注意事项

- 删除文件夹时，`imageFolderMap` 中的关联记录通过外键级联删除（需确认 `PRAGMA foreign_keys = ON`，已在 `AppDatabase` 中设置）
- `SEMANTIC_LABEL` 规则匹配采用精确词匹配（逗号分割后逐词比较），不使用 `contains` 子串匹配，避免「海边」误匹配「海边别墅」
