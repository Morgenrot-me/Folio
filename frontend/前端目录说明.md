# 前端架构说明

## 1. 目录作用
存放智能图片分类的全部前端代码，包含基于 Flutter 的跨平台界面和 Dart 业务逻辑代码。负责 UI 展示、用户交互及底层（本地/后端）调用。

## 2. 文件列表
本目录由 Flutter 标准结构生成，核心业务代码结构如下：
- `lib/main.dart`: 应用程序入口与 Providers 全局依赖挂载点
- `lib/core/`：核心驱动底座框架
  - `database/`: Drift 数据库配置、映射表定义与迁移脚本
  - `services/`: 无头守护进程后台服务（如负责执行 MediaStore 扫描和 TFLite 张量提取的服务）
  - `constants/imagenet_labels.dart`: ImageNet 1000 个物体名语义字典（已弃用，由 ML Kit 替代）
  - `constants/mlkit_zh_labels.dart`: ML Kit ImageLabeler 英文→中文静态翻译词典（400+ 标签）
  - `constants/label_normalizer.dart` [NEW]: 标签后处理归一化工具，同义词合并 + 黑名单过滤
  - `services/media_scanner_service.dart`: 调用原生系统权限拉取相册缩略图流
  - `services/feature_extractor_service.dart`: 双管线AI特征提取（ML Kit + Places365 TFLite），场景标签优先，调用 LabelNormalizer 后处理
  - `services/smart_folder_matcher_service.dart`: 规则树评估引擎，支持 IS_SCREENSHOT / HAS_TEXT / BLUR_SCORE / COLOR_WARMTH / SEMANTIC_LABEL 五种叶节点规则
  - `theme/app_theme.dart`: Material 3 全局深浅色动态托盘体系
- `lib/features/`：视图业务功能区块
  - `root/root_screen.dart`: 全局底部跨标签导航骨架
  - `home/home_screen.dart`: 核心仪表大盘与提取引擎触发视图
  - `gallery/gallery_screen.dart` [NEW]: 所有已索引相片的全局大画廊瀑布流视图
  - `gallery/image_detail_screen.dart` [NEW]: 单张相片的底层 AI 特征数据（如模糊度、文字标识、语义张量等）全透视面板
  - `folders/folders_screen.dart` [NEW]: 基于大模型的智能分类虚拟文件夹网格视图
  - `folders/create_folder_screen.dart` [NEW]: 新建/配置智能文件夹及其内嵌规则表单的向导 UI
- `pubspec.yaml`: 依赖与资源白名单配置文件

## 3. 技术说明
本项目采用 Flutter 和 Material Design 3 进行构建，通过 tflite_flutter 执行本地模型推理。

## 4. 使用示例
进入该目录下，通过以下命令启动调试：
```bash
cd frontend
flutter run
```

## 5. 扩展方式
- 新页面/功能模块应组织在 `lib/features/` 目录下。
- 新增依赖需要在 `pubspec.yaml` 中声明。
