# 数据库架构说明

## 1. 目录作用
统一维护项目的数据库架构说明及迁移文件。该目录聚焦于本地端(如 SQLite + drift)，及远端（如有）的表结构演进与抽象。

## 2. 文件列表
- (暂无配置文件，仅有说明文档)

## 3. 技术说明
在纯端侧，此目录将存放相应的表定义草图和数据库迁移语句（如 Room 或 drift 的 schema 历史文件）。

## 4. 使用示例
在对应的应用代码内执行迁移检测，此处负责存档 DDL 等。

## 5. 扩展方式
- 当增加类似 `users`, `custom_tags` 等表结构时，请将变更脚本存入该处。

## 6. 注意事项
- 所有的数据字段更改必须要在前后端代码中得到同步的体现。
- 文档需与代码结构的变动同步更新。
