# 智能图像分类工具 — 技术栈框架

> 创建时间：2026-03-24
> 版本：v0.1（规划阶段）

---

## 项目定位

一款安卓端**纯本地**的图像特征识别与分类工具。

核心理念：将 AI 提取的图像特征作为**第一等公民**，直接驱动文件夹分类逻辑，而非仅用于搜索。用户可通过"智能文件夹"的与/或/非特征逻辑实现自动分类，同时支持 Slidebox 风格的手动逐张整理。

---

## 开发框架

| 层级 | 选型 | 理由 |
|------|------|------|
| UI 框架 | **Flutter + Material Design 3** | 一套代码覆盖安卓 + 后续桌面端 |
| 推理框架 | **TensorFlow Lite (tflite_flutter)** | 安卓端最成熟的端侧推理方案，支持 GPU Delegate 加速 |
| 文字检测 | **ML Kit On-device Text Recognition** | 本地运行，判断图片是否含文字 |
| 后台调度 | **WorkManager** | 充电 + Wifi 条件下静默后台建库 |
| 媒体访问 | **MediaStore API** | 访问系统相册，无需复制原文件 |

## 特征提取层

### 特征类型总览

```
语义特征     MobileCLIP（Apple开源）
             → 输出 512 维语义向量
             → 覆盖：场景 / 物体 / 风格 / 人物

技术特征     多方案并行（纯本地）
             ├─ 截图检测     状态栏像素规则 + 宽高比检测（纯算法）
             ├─ 文字检测     ML Kit Text Recognition（Google本地模型）
             ├─ 模糊度评分   拉普拉斯方差（纯算法）
             └─ 颜色调性     HSV 直方图（纯算法）

聚类特征     感知哈希 pHash  → 64 位哈希，用于近似重复图片检测
             HDBSCAN 聚类    → 基于 512 维向量的语义相似聚类
```

### 单张图片处理流程

```
新图片进入系统
    │
    ├─ [并行] MobileCLIP → 512 维语义向量
    ├─ [并行] ML Kit    → 是否含文字 (bool)
    ├─ [并行] 截图检测   → 是否截图 (bool)
    ├─ [并行] pHash      → 64 位感知哈希值
    └─ [并行] 颜色/清晰度 → 若干 float 值
         │
         └─ 写入本地数据库（向量 + 技术特征元数据）
              │
              └─ 触发智能文件夹规则匹配 → 自动归档
```

## 存储层

| 存储内容 | 方案 | 说明 |
|---------|------|------|
| 语义向量 | SQLite（MVP 阶段） → Hnswlib | 万张以内 SQLite 余弦相似度足够；后期升级为 ANN 索引 |
| 技术特征元数据 | Room + SQLite | 截图标记、文字标记、模糊度、颜色向量等 |
| 智能文件夹规则 | Room + SQLite | 结构化存储 AND/OR/NOT 逻辑树 |
| EXIF 元数据 | Room + SQLite | 拍摄时间、GPS、设备等 |

## 性能参考（中端安卓设备）

| 操作 | 预估耗时 |
|------|---------|
| 单张 MobileCLIP 推理 | 50 ~ 150 ms |
| 单张技术特征提取 | < 10 ms |
| 单张 ML Kit 文字检测 | 30 ~ 80 ms |
| 万张图首次建库 | 约 15 ~ 30 分钟（后台静默） |
| 智能文件夹规则匹配查询 | < 50 ms |

## 隐私承诺

- 所有图像处理均在设备本地完成，**不上传任何图片或特征数据**
- AI 模型随 App 安装包或首次启动下载，后续完全离线运行
- 图片原文件不移动、不复制，仅读取元数据与像素用于特征提取
